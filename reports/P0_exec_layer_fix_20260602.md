## 一、问题定性 (全方位审计发现)

c023b59 声称"大球阈值 1.45→1.70 + 尾部共振豁免 + 常量单一真相来源 + 57场全回归通过",
但实证审计发现该 commit **只改了观察层一处**, 同一套大球判据的执行层和提示层全部漏改,
导致系统三层口径撕裂、175 场回归校准的核心收益在生产链路根本没生效。

### 撕裂全貌 (修复前)
| 层 | 函数 | 修复前阈值 | 后果 |
|---|---|---|---|
| 观察层 | `_total_goal_anchor_facts` | ✅ 1.70 + 尾部共振 + 常量 | c023b59 唯一改对的 |
| **执行层** | `_handicap_anchor_facts` (big_goal_injection) | ❌ 硬编码 `<=1.45`, **无尾部共振** | 真正强制并入 must_audit_scores 的闸门没生效 |
| **提示层** | `_cross_anchor_questions` | ❌ 硬编码 `<=1.45` | 给 AI 的交叉提问仍用旧阈值 |
| 描述层 | 读盘闸2 / Gemini终审 prompt | ⚠️ 口径未对齐 | 文案与执行口径矛盾 |

### 实际危害
1. **校准收益归零**: slope ∈ (1.45, 1.70] 的比赛, 观察层说"大球真信号", 执行层却按 1.45
   判为"单点诱盘假信号"不注入 — 召回 28.6%→75% 的核心增益在执行层根本没兑现。
2. **尾部共振对其目标 case 无效**: 巴西6-2/大阪6-1 类碾压大球, 尾部共振豁免只存在于观察层,
   真正决定 must_audit_scores 的执行层完全没有该逻辑, 照样漏掉。
3. **假绿盲区**: 旧 57 项回归无一覆盖大球注入逻辑, 所以执行层漏改还能"全绿通过"。

## 二、修复内容

### 1. 执行层 `_handicap_anchor_facts` (predict.py ~863)
- `_slope <= 1.45` → `_slope <= BIG_GOAL_SLOPE_THRESHOLD`
- 取 a6/a7, 新增 `_tail_resonance`, 注入条件改为 `(_slope_collapse or _tail_resonance)`
- 注入 payload 增加 a6/a7/trigger 字段, reason 动态标注触发原因

### 2. 提示层 `_cross_anchor_questions` (predict.py ~898)
- 三道分支阈值全部改引用 `BIG_GOAL_SLOPE_THRESHOLD`
- 新增"超大球尾部共振"独立提问分支 (与执行层口径一致)
- 大球分支补充【HIGH档弃精确】指令: HIGH档无任何比分>16%概率(175场实证), 不押精确比分,
  主推"大球方向+主/客大胜方向" — 落地研究A"HIGH档精确比分数学不可求"的结论

### 3. 描述层 prompt (predict.py 1186 / 3745)
- 读盘闸2: 档位描述补 `<=1.70→42%` + 尾部共振说明
- Gemini终审: `a5/a4<=1.45` → `a5/a4<=1.70 或超大球尾部共振(a6<=11/a7<=14)`

### 4. 新增回归测试 tests/test_big_goal_threshold_layers.py (12项)
专门堵假绿盲区, 锁死三层口径一致, 重点覆盖 slope ∈ (1.45, 1.70] 校准核心区间。
