# 2026-06 友谊赛窗口经验总结与最小升级

## 样本事实

本轮复盘集中在世界杯前/国际比赛日前后的热身赛、国际友谊赛。典型暴露：

- 强队/主队方向仍可能可读，但 **1-0 / 2-0 零封主线很脆**。
- 友谊赛下半场换人、试阵、保护主力，会把低比分胜局打散成 BTTS 或平局。
- 防平不能只防 `0-0 / 1-1`，开放战会把平局风险抬升到 `2-2`。
- 缺少官方首发、外部盘口时间序列、确认 sharp 时，不能把低赔主胜/客胜包装成精选。

## 周五4场实证片段

| 比赛 | 系统主比分 | 实际 | 复盘结论 |
|---|---:|---:|---|
| 新加坡 vs 中国 | 1-2 | 1-2 | 客胜 + BTTS + 3球完全兑现 |
| 斯洛伐克 vs 黑山 | 1-0 | 2-2 | 最大失误：低比分主胜零封被 BTTS 与 2-2 尾部击穿 |
| 匈牙利 vs 芬兰 | 2-0 | 2-1 | 方向对，零封失败；2-1 近邻应更高权重 |
| 加拿大 vs 爱尔兰 | 1-0 | 1-1 | 防平提示正确，但最终方向未翻转；友谊赛主队热身不一定追胜 |

硬口径：方向 2/4，比分 1/4；近邻比分 3/4。精选闸门未放行是正确的。

## 升级原则

本次不改 AI 终审方向/比分，不让本地代码成为第四个足球裁判；只把经验固化为：

1. 联赛 DNA：新增 `friendly_context`。
2. 风险闸门：识别友谊赛低比分零封陷阱。
3. 风险展示：强制补 `1-1 / 2-1或1-2 / 2-2` 近邻。
4. 行动层：触发时优先 `防平/观察`，不进入主推精选。

## 新增规则

若满足：

- league 为 `国际赛/国际友谊/友谊/热身/friendly`；
- final_direction 为 home 或 away；
- predicted_score 为低总进球零封路径，如 `1-0 / 2-0 / 0-1 / 0-2`；
- 且无 web/首发确认，或 BTTS=no，或存在平局/高分尾部信号；

则触发：`prematch_v2_friendly_clean_sheet_trap_guard`。

触发后：

- 不改 `predicted_score` 和 `final_direction`；
- 补充 `friendly_clean_sheet_fragility / friendly_btts_late_goal_risk / friendly_2_2_draw_tail`；
- 补充风险比分：
  - 主胜零封：`1-1`, `2-1`, `2-2`
  - 客胜零封：`1-1`, `1-2`, `2-2`
- selection layer 优先进入 `防平`。

## 后续大样本验证

这只是友谊赛专项保护，不代表正赛也同权重。正赛开始后继续跟踪：

- 友谊赛低赔强队零封预测的 BTTS 失效率；
- `1-0/2-0 -> 1-1/2-1/2-2` 的近邻迁移率；
- 是否需要把该规则从友谊赛扩展到杯赛/低战意联赛末轮。
