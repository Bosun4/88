## 2. 根因定位(只读核验)

### 2.1 终审是 prompt 驱动,本地不做足球决策
- v20.6 已拆除本地伪裁判:`protocol_downgrade_recommendation_only`(predict.py:3367)注释明确 "no longer locally caps AI recommendation tiers ... lets the AI final referee own tier"。
- 因此 203 回锚 1-1 **不是某段 Python 打分造成**,成因在「证据包呈现 + 终审协议提示词」两层。

### 2.2 证据包原料齐全,中立呈现(predict.py:957)
```
"movement": {"change": ..., "coding_note": "change_is_direction_code=true 时,-1=降水/下降,0=不变,1=升水/上升。"}
```
`movement.change`(降水/升水)、`correct_score_odds`、`lottery_market_1x2`、`public_vote` 全部作为事实喂给 AI。风险层正是据此正确探测到 0-1 客胜降水——**原料没问题**。

### 2.3 终审协议第4条只是"描述性指引",缺硬否决(predict.py:3333-3334)
现协议原文:
```
- 如果大众极热某方向、但临场赔率反向大降水或不降反升,必须警惕"正诱造热/大热必死"陷阱,主动下调推荐等级;
- 如果某方向临场庄家急剧降水保护,寻找被隐藏的资金入场。
```
→ 鼓励 AI **看**降水,但**没有禁止**"当客胜降水已确认、客胜最低比分进低赔前二时,仍因'最低正确比分恰好是平局'而回锚平局"。裁判有了背离意识,却保留往平局缩的自由。

### 2.4 203 的自相矛盾(终审日志实证)
- 风险标签已把 **0-1 列为头号风险**:"客队一球偷胜 - 客胜方向有降水事实,0-1为客胜最低比分9.0"
- 另挂 `away_steam_candidate`、`weak_market_direction_conflict`、`one_goal_margin_swing`
- 终审理由却是:"正确比分最低为1-1,总进球最低2球,相邻审计后1-1无更低竞争者 → 落平局1-1"

**定性**:泊松/1-1锚弃用后的**协议层惯性残留**。不再有显式锚,但"最低正确比分恰好是平局"时,裁判层仍默认回锚平局,把已确认的资金降水背离压了下去。

## 3. 改造方案(单点、低风险、纯 prompt)

在终审协议第4条后新增 **4b 背离否决硬约束**,把"已确认资金降水方向"升格为对平局锚的否决权。

### 3.1 拟新增条款(草案)
> 4b.【确认降水否决平局锚】当某一非平局方向同时满足:(a) 临场赔率已确认降水(change=-1);(b) 该方向最低正确比分进入全场低赔前二;(c) 出现 market_direction_conflict——此时你**禁止**仅因"最低正确比分恰好是平局"而回锚平局。确认的资金降水方向优先级高于静态最低正确比分锚。若仍坚持平局,必须在 final_referee_analysis 显式说明为何无视已确认降水,否则视为追低赔锚失职。

### 3.2 改动范围
- 文件:`scripts/predict.py`,仅终审协议字符串(约 3334 行后插入一条)。
- **不碰任何 Python 决策逻辑**,完全守"本地不改足球方向/比分"军规。
- 触发条件三者同时成立才生效。

### 3.3 误伤评估(对照正例 204)
- 204 哥伦比亚:主胜方向**无降水冲突**(主胜1.12、比分簇前六全主胜、无 market_direction_conflict),(c) 不成立 → 4b 不触发 → 2-0 锚仍可信。
- 结论:新条款只在"确认降水 + 客胜低赔前二 + 冲突"三条件交集触发,**不误伤资金同频的正例**。
