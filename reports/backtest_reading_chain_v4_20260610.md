## 二、诚实结论（不修饰）

1. **v4 机械骨架整体没有跑赢，不具备独立落地资格。** 方向与纯低赔基准打平（47.6%），比分与进球带都输给线上系统。按"跑不赢基准即切除"的标准，**链条的机械部分不能作为本地算法落地**——这同时再次印证：把博弈判断机械化（哪怕是我自己设计的链）就是另一种泊松。
2. **但链条中两个子信号被数据证实有效：**
   - **分位簇塌缩信号（Step2 核心）**：触发 41 场，实际 4+ 球 44%，对比全样本 4+ 基准率 32% → **+12 个百分点提升**。"a4 与 a5 同步处于联赛 p25 以下"是真信号。
   - **单点诱盘拦截**：a4 低但 a5 不跟的场次触发 2 场，实际 4+ 球 0%——拦截方向正确（样本太小，仅作方向参考）。
3. **一个连带发现：线上系统方向命中 44.7%，跑输纯低赔基准 47.6% 约 3 个百分点。** 系统的 AI 读盘在这 176 场上对方向是负贡献的（比分命中 15.9% 优于机械链，说明 AI 的价值在比分形状而非方向）。
4. change 缺失的 147 场中 v4 方向 48% vs 系统 44%；change 可用的 23 场两者持平 48%——走向数据在本样本里未体现增量（样本 23 场太小，不下结论）。

## 三、对 v4 方案的修正

- **砍掉**: Step3 的"候选集内取最低赔存活"机械规则（比分命中 12.4% < 系统 15.9%，证伪）。比分收敛必须留给 AI 做逆向淘汰，机械求交只提供候选集边界。
- **保留并升级为事实注入**: 联赛分位表 + 簇塌缩/单点诱盘标记（44% vs 32% 基准，证实）。物理层只编译"本场 a3-a6 的联赛分位位置 + collapse 标记"，判断权归 AI。
- **维持研究态**: 真正的验收测试应是**新旧 prompt 的真实 AI A/B**（机械骨架测不出 prompt 引导对 AI 推理的影响）。建议抽 20-30 场已有赛果场次，旧 prompt vs 注入 v4 链的新 prompt 各跑一遍三 AI 链路对比。此步消耗真实 API 调用，待批准后执行。

## 四、一句话

机械骨架不配上线（证伪），分位塌缩信号是真金（+12pp，证实），比分判断的价值在 AI 不在机械——v4 落地形态收缩为"物理层分位事实注入 + prompt 四步链引导"，最终验收需真实 AI A/B，等批准。
